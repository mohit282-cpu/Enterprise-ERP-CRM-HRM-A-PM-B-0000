## 🌟 Project Description

**Sovryx OS** is a comprehensive, centralized Enterprise Business Operating System developed by **Sovryx Tech**. It is designed from the ground up to replace the fragmented landscape of multiple software products (CRM, ERP, Accounting, HR, etc.) with one unified, blazing-fast, and intelligent platform.

Built with performance, security, and scalability in mind, Sovryx OS empowers modern IT companies, software agencies, and enterprises to streamline their daily operations, automate tedious tasks, and gain unparalleled insights into their business metrics.

## 📐 Architecture

### MVC Pattern
Sovryx OS adheres to a strict **Model-View-Controller** design pattern:
- **Models:** Handle business logic, database interactions, and data validation.
- **Views:** Handle presentation logic using raw PHP templating combined with HTML/CSS/JS.
- **Controllers:** Act as the glue between Models and Views, handling HTTP requests and returning appropriate responses.

### REST API Structure
Our API follows RESTful principles, returning standard JSON responses. 
- **Endpoint Structure:** `/api/v1/resource`
- **Methods:** `GET`, `POST`, `PUT`, `PATCH`, `DELETE`
- **Authentication:** Bearer Tokens / API Keys.

### Folder Structure
```text
sovryx-os/
├── app/
│   ├── Controllers/       # Application controllers
│   ├── Models/            # Database models and business logic
│   ├── Views/             # HTML templates and UI components
│   ├── Helpers/           # Utility functions (formatting, validation)
│   ├── Middlewares/       # Authentication, CSRF, Role checks
│   └── Services/          # Complex business logic (AI, WhatsApp API)
├── config/                # System configuration files (DB, Mail, App)
├── public/                # Document root
│   ├── index.php          # Front controller (Entry point)
│   ├── assets/            # CSS, JS, Images, Fonts
│   └── uploads/           # User generated content
├── routes/                # Application route definitions (Web & API)
├── database/              # Migrations and seeders
├── vendor/                # Composer dependencies
├── .env.example           # Environment variables template
├── composer.json          # Dependency definitions
└── README.md              # Project documentation
```

## 🗄 Database Design Overview

The database is highly normalized (3NF) ensuring data integrity and optimized query performance.

* **Core Tables:** `users`, `roles`, `permissions`, `role_user`, `settings`
* **CRM/Sales:** `clients`, `leads`, `invoices`, `invoice_items`, `payments`, `quotations`
* **HRM:** `employees`, `departments`, `attendance`, `payroll`, `leaves`
* **Projects:** `projects`, `tasks`, `timesheets`, `milestones`
* **IT Specific:** `domains`, `hostings`, `servers`, `ssl_certificates`
* **Support:** `tickets`, `ticket_replies`, `knowledge_base`

*Foreign keys are strictly enforced with cascading rules tailored to business logic (e.g., deleting a client archives their data rather than deleting financial records).*

## ⚙️ Installation Guide

### Requirements
- **OS:** Ubuntu 20.04/22.04 LTS (Recommended) or any modern Linux distribution / Windows (XAMPP/Laragon).
- **Web Server:** Apache 2.4+ or Nginx.
- **PHP:** Version 8.3 or higher.
- **PHP Extensions:** `pdo_mysql`, `mbstring`, `curl`, `gd`, `xml`, `zip`, `openssl`.
- **Database:** MySQL 8.0+ or MariaDB 10.4+.
- **Composer:** Latest version.

### Environment Variables
Rename the `.env.example` file to `.env` and configure your local settings:
```ini
APP_NAME="Sovryx OS"
APP_ENV=production
APP_URL=https://yourdomain.com
APP_DEBUG=false

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=sovryx_os
DB_USERNAME=root
DB_PASSWORD=your_secure_password

MAIL_MAILER=smtp
MAIL_HOST=smtp.mailtrap.io
MAIL_PORT=2525
MAIL_USERNAME=your_username
MAIL_PASSWORD=your_password
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=hello@sovryxtech.com

WHATSAPP_API_KEY=your_api_key
OPENAI_API_KEY=your_openai_key
```

### Composer Installation
Navigate to your project root and install the dependencies:
```bash
composer install --optimize-autoloader --no-dev
```

### Database Setup
1. Create a new MySQL database named `sovryx_os` (or match your `.env`).
2. Import the schema/migrations:
```bash
# Example command if using a custom CLI tool, or import SQL dump
php sovryx migrate
php sovryx db:seed
```

### Folder Permissions
Ensure the web server has write access to necessary directories:
```bash
chmod -R 775 public/uploads/
chmod -R 775 storage/
chown -R www-data:www-data public/uploads/ storage/
```

## 🚀 Running the Project

### Build Instructions
For local development, you can use PHP's built-in server:
```bash
cd public
php -S localhost:8000
```
Navigate to `http://localhost:8000` in your browser.

### Deployment Guide

#### Apache Configuration
Create a new VirtualHost (`/etc/apache2/sites-available/sovryx.conf`):
```apache
<VirtualHost *:80>
    ServerName yourdomain.com
    DocumentRoot /var/www/sovryx-os/public

    <Directory /var/www/sovryx-os/public>
        AllowOverride All
        Require all granted
    </Directory>

    ErrorLog ${APACHE_LOG_DIR}/sovryx_error.log
    CustomLog ${APACHE_LOG_DIR}/sovryx_access.log combined
</VirtualHost>
```

#### Nginx Configuration
Create a server block (`/etc/nginx/sites-available/sovryx`):
```nginx
server {
    listen 80;
    server_name yourdomain.com;
    root /var/www/sovryx-os/public;
    index index.php index.html;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/var/run/php/php8.3-fpm.sock;
    }
}
```

## 🔄 System Workflows

### Authentication Flow
1. User submits login credentials.
2. Controller sanitizes input and verifies against the hashed password in the DB.
3. Upon success, a secure Session ID is generated.
4. User roles and permissions are fetched and stored in the session.
5. CSRF tokens are generated for subsequent POST requests.

### Role Based Permission System
Sovryx OS uses a granular RBAC system. A user can have multiple roles, and each role has specific permissions (e.g., `create_invoice`, `view_reports`, `delete_user`). Middleware intercepts requests to verify if the active user possesses the required permission string.

### Accounting & Invoice Workflow
1. **Proposal:** AI generates a proposal -> Sent to Client.
2. **Approval:** Client approves proposal online.
3. **Invoice Generation:** Proposal converts to Invoice instantly.
4. **Payment Tracking:** Payment received -> Logs into accounting ledger -> Updates Cash Flow -> Sends WhatsApp/Email receipt.

### Project & Task Workflow
1. **Initiation:** Project created -> Assigned to a team.
2. **Milestones:** Tasks mapped to milestones on the Kanban board.
3. **Time Tracking:** Developers track time on tasks.
4. **Billing:** Billable hours automatically synced to the client's next invoice.

### Hosting & Domain Workflow (For IT Companies)
1. System checks expiry dates via Cron Job daily.
2. 30, 15, and 3 days before expiry, automated email/WhatsApp reminders are sent to the client.
3. If unpaid, service status updates to "Suspended", triggering an internal ticket.

### AI Workflow
Utilizing OpenAI's APIs, the AI Assistant analyzes context (e.g., client history, project details) to generate tailored emails, draft scopes of work, and summarize long support tickets.

## 🔒 Security & Performance

### Security Features
- **Strict PDO Parameterization:** Absolute prevention of SQL Injection.
- **XSS Filtering:** All user outputs are escaped using `htmlspecialchars`.
- **CSRF Tokens:** Required on every state-changing request.
- **Rate Limiting:** Login endpoints are protected against brute-force attacks.
- **Audit Logging:** Every critical action (delete, financial update, permission change) is logged with User ID, IP address, and timestamp.

### Performance Optimization
- **Caching Strategy:** Database query caching for configuration and permissions. Static assets are versioned and browser-cached.
- **Autoloading:** Composer's optimized PSR-4 autoloader ensures files are only loaded when needed.
- **Database Indexing:** Crucial columns (email, client_id, status) are indexed for lightning-fast lookups.

### Backup Strategy
Built-in automated database dump and file backup utility that can securely push encrypted ZIP files to AWS S3, Google Drive, or a local storage volume on a cron schedule.

### Logging System
Comprehensive error logging using Monolog (or similar) separated into daily files: `system.log`, `database.log`, `api_requests.log`.

## 🤝 Contributing Guide

We welcome contributions to Sovryx OS! 

### Coding Standards
- We follow **PSR-12** coding standards.
- Ensure strict typing (`declare(strict_types=1);`) in all PHP files.
- Document classes and methods using standard PHPDoc blocks.

### Git Workflow & Branch Strategy
- `main` : Production-ready code.
- `develop` : Integration branch for ongoing development.

1. Fork the repository.
2. Create your feature branch (`git checkout -b feature/AmazingFeature`).
3. Commit your changes (`git commit -m 'Add some AmazingFeature'`).
4. Push to the branch (`git push origin feature/AmazingFeature`).

## ❓ FAQ & Troubleshooting

<details>
<summary><strong>How do I reset an administrator password?</strong></summary>
You can run the CLI command `php sovryx user:reset-password admin@yourdomain.com` or directly update the hash in the database if CLI is unavailable.
</details>

<details>
<summary><strong>My invoices are not generating PDFs. Why?</strong></summary>
Ensure the `storage/` directory has `775` permissions. Dompdf requires write access to generate temporary font and PDF files.
</details>

<details>
<summary><strong>Emails are failing to send.</strong></summary>
Check your `.env` file to ensure the SMTP credentials are correct. You can view the `system.log` to see the exact PHPMailer exception.
</details>

### Known Issues
- Large datasets in the Accounting Ledger can cause minor UI lag on mobile devices. (Fix planned in v1.2 with virtualized scrolling).
- WhatsApp API session may occasionally disconnect requiring manual re-authentication via the settings panel.
